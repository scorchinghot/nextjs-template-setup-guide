‚öôÔ∏èGetting Started

Open a terminal and go to the folder:

cd path-to-your-template/seme (seme is example could be anything like illucom or relstate)

Or just drag the seme folder into VS Code and hit Ctrl + ~ to open the terminal.

Now run:

npm install

If that fails, try:

npm install --force

It just skips version issues. No big deal.

Then start the dev server:

npm run dev

Or try next dev (check important.txt for details)

Now open your browser and go to:
http://localhost:3000

You're in.

If anything breaks or acts weird, check important.txt ‚Äî it'll save you.

üì¶Static Export (aka turning this into HTML-ish)

Want to turn this into a static site you can host anywhere? Here's how:

  1. Open your next.config.js or next.config.ts and paste this in:
import type { NextConfig } from "next"; const nextConfig: NextConfig = { eslint: { ignoreDuringBuilds: true, }, images: { unoptimized: true, }, output: 'export', trailingSlash: true, }; export default nextConfig;
  1. In your terminal:
npm run build

It'll create an out/ folder.
Upload that to Netlify, GitHub Pages, Vercel ‚Äî whatever.

If it doesn't work, it's probably because some features can't be statically exported.
Check important.txt ‚Äî I'll let you know if this template is fully static-safe or not.

üåêSSR Deployment (if static export won't cut it)

If your site needs dynamic stuff (forms, auth, etc), use Vercel:
